Architecture

Odyssey uses a simple client-server architecture with no build tooling. The Node.js server handles file I/O and API proxying, while the browser handles all AI communication, game logic, and rendering.

Key Design Decision

API keys and AI calls are managed entirely client-side. The server never touches API credentials — it only stores game saves, serves static files, and proxies xAI requests to bypass CORS. This makes the server stateless and simple.

Request Flow

FLOW
Browser (setup.js / game.html)
  │
  ├─→ Direct HTTPS to AI Provider (OpenRouter, Google AI, etc.)
  │     └─→ JSON Schema–enforced response
  │
  ├─→ /api/xai-proxy/* → server.js → https://api.x.ai/* (CORS bypass)
  │
  └─→ /api/* (save-game, load-game, presets, images)
        └─→ server.js → filesystem (./games/, ./presets/)

Server (server.js)

The server is a single-file Node.js HTTP server with zero external dependencies for serving (only @openrouter/sdk is listed in package.json). It uses native http, fs, and path modules.

Key Responsibilities

  • Static file serving — Serves HTML, CSS, JS, images with correct MIME types and cache-busting support
  • Game save management — CRUD operations on ./games/{id}/ directories
  • Character presets — Save, list, and delete preset JSON files in ./presets/
  • Image handling — Download, save, and serve player character images
  • xAI CORS proxy — Forwards requests from /api/xai-proxy/* to https://api.x.ai/*

⚠️ Security Hardening

The server includes directory traversal prevention (path normalization + __dirname boundary checking), input ID sanitization (numeric-only), and SSRF protection on image downloads (HTTPS-only URLs). See the Security Notes section.

API Reference

All endpoints are served on port 3001 by default. The server binds to 0.0.0.0 for LAN access.

Game Management

MethodEndpointDescription
GET/api/list-gamesReturns JSON array of game folder names
GET/api/load-game?id={id}Loads all JSON files from a game directory. ID is numeric-only sanitized.
POST/api/save-new-gameCreates a new numbered game directory with worldinfo, player, gamestate, scenario, and default ledger files.
POST/api/update-gameUpdates gamestate, chat history, NPC ledger, locations ledger, and/or summary for an existing game.

Character Presets

MethodEndpointDescription
GET/api/list-presetsReturns array of all preset JSON objects from ./presets/
POST/api/save-presetSaves a preset. Filename is sanitized from presetName.
POST/api/delete-presetDeletes a preset file by name.

Images & Proxy

MethodEndpointDescription
POST/api/download-imageDownloads an image from HTTPS URL or saves a base64 data URI to game folder.
POST/api/update-imageUpdates the player image URL reference for a game.
GET/api/get-base-image?id={id}Returns the player base image as a base64 data URI.
POST/api/shutdownGracefully shuts down the server process.
*/api/xai-proxy/*Forwards requests to https://api.x.ai/ with auth headers intact. Used for CORS bypass.

Page Flow

The application is a multi-page app (MPA) with client-side state managed via localStorage and URL query parameters.

NAVIGATION
Welcome.html  (cinematic intro → content warning)
    ↓ Agree
mainmenu.html (New Game / Load Game / Settings / Presets / Info)
    ├─→ game.html?action=new     → setup.js wizard → gameplay
    ├─→ game.html?action=load&id=N → load save → gameplay
    ├─→ settings.html?from=mainmenu
    ├─→ presets.html

    ├─→ settings.html?...&from=game → returns to game.html
    └─→ mainmenu.html (via Main Menu button)

Setup Wizard (setup.js)

The setup.js file (3200+ lines) manages the complete new game creation flow. It runs inside game.html when ?action=new is in the URL.

Five Phases

PhaseNameDescription
1World SetupChoose Real Universe or Custom. Custom worlds have 8 fields (name, era, setting, tone, magic, rules, dangers, factions). Presets auto-fill known IPs.
2Player Creation12+ fields including attribute sliders (athleticism, intelligence), AI-generated text for appearance, personality, backstory, family, friends, inventory.
3Starting ScenarioUser writes or AI generates the opening scene description.
4AI SummaryAI processes all world + player data into a coherent summary. Optional character image generation.
5Game LaunchSaves all data via /api/save-new-game, transitions to the live chat game view.

Field Types

  • text / number — Simple input with optional 🎲 AI auto-generate
  • select — Dropdown (used for preset selection, gender)
  • attribute — Snapping slider with labeled markers + "Define it myself" toggle
  • ai — Textarea with AI draft → review → approve/edit/retry flow
  • date — Calendar-aware (Gregorian, Star Wars BBY/ABY, LOTR Ages, etc.)

Key Function: buildFetchPayload()

Constructs the OpenAI-compatible API payload. Handles provider-specific quirks (xAI omits penalty params, LM Studio uses json_schema instead of json_object).

Game Engine

During gameplay, the chat interface in game.html sends user actions to the AI with the full game context. The AI responds with structured JSON matching the gameOutputSchema.

UI Layout (3-Column Grid)

  • Left Column — Player stats (health, money, hunger, thirst, energy), player image, Main Menu button, Player Menu button
  • Center Column — Chat interface (message history + input area). Max width 850px, auto-scrolling.
  • Right Column — In-game clock (supports 5 calendar systems), Settings button, Game Codex button

Modals

  • Player Menu — Left side: player profile (image + attributes). Right side: inventory grid with auto-scaling item names.
  • Game Codex — Left side: NPC Ledger. Right side: Location Ledger. Both auto-populated from AI responses.

How Themes Work

Each theme defines a vars object of CSS custom properties that get applied to :root. All UI components reference these variables (e.g., var(--bg-secondary), var(--accent-color)). Canvas themes additionally create a <canvas id="bg-canvas"> element with requestAnimationFrame loops.

Adding a New Theme

theme.js
// Add to the presetThemes object:
'mytheme': {
    name: 'My Theme',
    type: 'standard',  // or 'image', 'canvas-stars', 'canvas-matrix'
    vars: {
        '--bg-main': '#...',
        '--bg-secondary': '...',
        '--text-main': '...',
        '--text-heading': '...',
        '--accent-color': '...',
        // ... all 12 CSS variables
    }
}

UI Components (ui-components.js)

A custom UI component library that replaces native HTML elements with styled, accessible versions.

Custom Select Dropdown

createCustomSelect(nativeSelect, opts) — Hides the native <select> and creates a fully styled dropdown with optional search filtering. Supports <optgroup> labels and per-option font previews.

Custom Number Spinner

createCustomNumberInput(nativeInput) — Wraps <input type="number"> with up/down arrow buttons that support click-and-hold for rapid increment.

Auto-Initialization

USAGE
// Call after DOM is ready:
initCustomUIComponents(document.body, {
    searchSelects: ['model-select', 'font-select']  // IDs to enable search
});

// Refresh after programmatic option changes:
refreshCustomSelect('model-select');

Game State Schema

The AI is instructed to respond with JSON matching this schema. It is enforced via the response_format.json_schema parameter on each API call.

gameOutputSchema
{
  "time": {
    "hour": integer,
    "minute": integer,
    "period": "AM" | "PM",
    "dayOfWeek": string,
    "day": integer,
    "month": integer,
    "year": integer,
    "era": string,
    "calendarType": string
  },
  "textoutput": string // Markdown-formatted narrative,
  "inventory_changes": [{ action, name, newName, description }],
  "location_changes": [{ name, description }],
  "npc_changes": [{ name, status_or_history }],
  "stats": { health, money, hunger, thirst, energy }
}

💡 Inventory Actions

The inventory_changes array supports three actions: "add" (new item), "remove" (drop/use item), and "update" (rename or change description via newName).

Contribution Guide

Getting Started

  1. Fork the repository on GitHub
  2. Clone your fork and run npm install
  3. Create a feature branch: git checkout -b feature/my-feature
  4. Make your changes and test locally with npm start
  5. Push and open a Pull Request

Code Conventions

  • No build tools or transpilers — keep it vanilla HTML/CSS/JS
  • Use CSS custom properties (defined in theme.js) for all colors
  • Prefix all localStorage keys with jsonAdventure_
  • Sanitize all user inputs before filesystem operations
  • Game IDs are numeric-only — enforce with .replace(/[^0-9]/g, '')
  • Preset filenames are sanitized: .replace(/[^a-z0-9]/gi, '_').toLowerCase()

Testing Tip

The server prints LAN IPs on startup. Test on mobile by connecting to the same network and navigating to the printed IP address.

Security Notes

The server implements several security hardening measures:

🛡️ Directory Traversal Prevention

Static file paths are normalized with path.normalize() and verified to be within __dirname. Requests resolving outside the project root receive a 403 Forbidden.

🛡️ Input Sanitization

Game IDs are stripped to digits-only (/[^0-9]/g). Preset names are sanitized to alphanumeric + underscores. This prevents path injection via crafted IDs.

🛡️ SSRF Protection

The image download endpoint only allows https: protocol URLs, preventing attackers from using the server as an open proxy to internal services.

🛡️ CORS Policy

No Access-Control-Allow-Origin header is set, relying on browser Same-Origin policies. The xAI proxy strips incoming CORS headers from upstream responses.
